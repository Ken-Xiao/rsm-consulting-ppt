# Action Derivation Chain

用于确保行动地图、建议页、12 个月路线图和管理层授权事项不是“建议清单”，而是从前文诊断自然推导出的因果链终点。

## Core Principle

每个行动项必须能回链到：

```text
矛盾/风险/机会 -> 根因 -> 改善路径 -> 行动项 -> 指标/责任/时点
```

如果一个行动项无法回链前文证据，应删除、降级为“待验证事项”，或补充诊断页。

## Action Derivation Schema

```json
{
  "actions": [
    {
      "recommended_action_id": "act_001",
      "action_title": "建立风险调整收益月度复盘机制",
      "linked_contradiction_id": "ct_001",
      "linked_root_cause": "风险定价不足导致账面 ROA 与 RAROC 背离",
      "improvement_path": "从规模导向转为风险调整收益导向",
      "owner": "风险管理部 / 计划财务部",
      "timeline": "0-3 个月",
      "kpi": ["RAROC", "不良生成率", "风险成本"],
      "trigger": "连续两个季度低于同业中位",
      "evidence_pages": ["P15", "P16", "P38"]
    }
  ]
}
```

## 12-Month Roadmap Rules

三阶段行动地图建议采用：

- `0-3 个月`：口径统一、数据治理、监控阈值、责任机制。
- `3-6 个月`：试点验证、产品/资产/客户组合调整、专项治理。
- `6-12 个月`：制度固化、资源配置、考核联动、复盘迭代。

每个阶段最多 3-4 个行动项。每个行动项必须包含：

- 动作对象。
- 责任主体或协同部门。
- 验收指标。
- 触发条件或时间点。
- 回链的矛盾、风险或机会。

## Recommended Wording

- `建议由 [责任主体] 在 [时间] 前完成 [动作]，并以 [指标] 作为验收标准。`
- `该行动对应前文 [矛盾/风险]，目的在于通过 [改善路径] 降低 [影响]。`
- `若 [前提] 不成立，则该行动应降级为试点或待验证事项。`

## Failure Signals

- 行动项只写“加强、优化、关注、提升”。
- 没有责任方、时间或指标。
- 行动项和前文问题没有明确关系。
- 行动地图出现前文未证明的新判断。
