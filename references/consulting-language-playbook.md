# Consulting Language Playbook

用于把中文金融 PPT 的表达压缩为专业咨询口径。写标题、副标题、结论条、建议和风险提示时读取本文件，并与 `language-discipline.md` 同时使用。

## Writing Principles

咨询 PPT 语言应满足：

- 判断清晰：标题说结论，不说主题。
- 证据匹配：措辞强度不能高于证据强度。
- 口径明确：涉及数字、比较、估值、风险时说明范围或前提。
- 短句优先：减少修饰词、过程词和泛化动词。
- 面向决策：尽量说明含义、取舍或下一步。

## Title Formula Library

优先使用以下标题句式。根据证据强度选择“显示 / 指向 / 初步显示 / 在……假设下”。

### 1. 结构分化

- `A 整体改善，但 B 与 C 的贡献分化扩大`
- `A 的增长主要来自 B，C 的支撑作用仍需验证`
- `A 与 B 出现背离，提示总量指标不足以解释经营质量`

### 2. 驱动归因

- `A 变化主要由 B 驱动，而非 C 的持续改善`
- `B 是 A 改善的主要贡献项，但其可持续性受 C 约束`
- `A 的短期改善来自 B，长期修复仍取决于 C`

### 3. 同业比较

- `A 位于样本前段，但 B 指标低于同业中位`
- `A 在规模指标上领先，效率和质量指标尚未同步改善`
- `相较同业，A 的差距集中在 B 和 C 两个环节`

### 4. 风险与边界

- `A 风险暴露已具备可识别特征，但损失水平仍取决于 B`
- `在 C 前提下，A 的可执行性较高；若 C 不成立，需要调整方案`
- `A 的判断边界主要来自 B 口径差异和 C 数据缺口`

### 5. 行动建议

- `下一步应优先验证 A，再决定 B 的资源投入`
- `建议将 A 作为短期抓手，将 B 作为中期能力建设方向`
- `若维持当前趋势，A 将成为 B 的主要约束，应提前设置监控指标`

## Subtitle Formula

副标题必须连接主标题和页面主体。可使用以下结构：

```text
基于 [样本/期间/口径]，通过 [方法/图表] 解释 [本页证明对象]，并为下一页 [下钻方向] 提供依据
```

示例：

- `基于 7 家上市险企 2024-2025 年披露口径，将净利润拆为承保、投资、所得税和其他项，识别增长来源差异`
- `在确认整体利润改善后，本页进一步比较不同公司保险服务收入与承保利润的匹配关系`
- `由于样本披露口径存在差异，本页仅使用可比字段判断费用效率的相对位置`

## Assertion Strength

| Evidence strength | Allowed words | Avoid |
|---|---|---|
| 强数据/原始文件 | `显示`、`表明`、`排名为`、`已形成` | `可能` 过度弱化 |
| 多证据交叉 | `指向`、`支持判断`、`较可能` | `证明`、`决定性` |
| 单一数据或推断 | `初步显示`、`可能`、`需进一步验证` | `显著`、`核心原因` |
| 情景测算 | `在……假设下`、`若……成立`、`情景显示` | 写成事实 |
| 专家判断 | `项目组判断`、`建议优先验证`、`建议纳入后续监控` | `必然`、`一定`、单独使用 `建议关注` |

## Compression Rules

默认长度：

- 主标题：24-36 个中文字符优先；复杂页不超过 48 个中文字符。
- 副标题：1-2 行；不超过 55 个中文字符优先。
- 结论条：2-3 句；每句不超过 35 个中文字符。
- 卡片标题：8-14 个中文字符。
- bullet：每条 18-32 个中文字符；超过 40 个中文字符应拆分或改写。

优先删除低信息词：

- `进一步`、`持续`、`不断`、`有效`、`全面`、`系统性`、`充分`、`较好`
- `具有重要意义`、`值得关注`、`有待加强`、`赋能`、`夯实`
- `通过……的方式来……` 改为直接动词

示例：

- 弱：`公司需要进一步持续加强投资端能力建设`
- 强：`投资收益波动扩大，需单独设置收益率和久期监控指标`

## Action Language Rule

客户交付中的建议必须可执行。禁止单独使用以下表达：

- `建议关注`
- `持续优化`
- `进一步加强`
- `强化管理`
- `提升能力`
- `完善机制`

必须改写为：

```text
建议 [责任主体/对象] 在 [时间/前提] 下，对 [指标/事项] 执行 [动作]，并以 [阈值/结果] 作为判断标准。
```

示例：

- 弱：`建议持续关注承保质量`
- 强：`建议将综合成本率、赔付率和费用率设为月度监控指标，连续两个季度偏离同业中位时启动专项复盘`
- 弱：`建议加强投资风险管理`
- 强：`建议按资产类别拆分投资收益率和久期敞口，单独识别利率变动对净利润的敏感性`

## Executive Rewrite Test

客户交付版的标题、结论条和建议必须通过“会议纪要测试”：

- 是否能被客户直接放入会议纪要、管理层汇报或决议附件。
- 是否能明确说明对象、动作、依据和边界。
- 是否避免“方向正确但不能执行”的泛化表达。

改写模板：

| Weak | Client-ready |
|---|---|
| `建议关注利润质量` | `建议以承保利润、投资收益率和费用率作为利润质量三项跟踪指标` |
| `需加强资产质量管理` | `建议对关注类贷款迁徙率连续两个季度上行的分行启动专项复盘` |
| `应优化资本使用效率` | `建议优先压降低收益高资本占用资产，并按季度跟踪 RWA 密度变化` |
| `进一步完善风险控制机制` | `建议为方案落地设置触发阈值、责任部门和月度监控表` |

## Sensitive Expression Replacement

| Avoid | Use |
|---|---|
| `表现较差` | `低于样本中位数 X` |
| `风险很大` | `风险暴露需要进一步量化` |
| `不可行` | `在当前约束下可执行性不足` |
| `必须` | `建议优先`、`需要`、`应在……前提下` |
| `严重` | `偏离样本中位数 X`、`超过阈值 X` |
| `明显好转` | `较上期改善 X` |
| `大幅下降` | `下降 X pct / X bp` |
| `核心原因` | `主要驱动之一`、`当前数据支持的主要解释` |

## Language QA

交付前抽查每页：

- 标题是否是判断，而不是主题。
- 标题的判断强度是否匹配证据强度。
- 副标题是否说明样本、期间、口径或本页证明对象。
- 是否存在没有量化参照的形容词。
- 是否存在“建议关注”但没有说明关注什么指标、阈值或动作。
- 是否将情景测算写成事实。
- 是否使用了未注解的英文缩写。
- 客户会不会把这句话直接放进会议纪要或决议材料；如果不能，压缩并补充边界。
