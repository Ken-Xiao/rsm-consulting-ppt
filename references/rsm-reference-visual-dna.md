# RSM Reference Visual DNA

用于把生成 PPT 的视觉效果对齐以下两份参考文件：

- `2025年上市保险公司新准则财务结果分析_final.pdf`
- `final-20251031 商业银行及保险公司不良处置行业实践分享-1103.pdf`

本文件是视觉复刻标准。生成、优化或 QA 时，如果用户要求“按参考 PDF 风格”，优先读取本文件。

## Two Reference Modes

### Mode A: Insurance Results Report

对标 `2025年上市保险公司新准则财务结果分析_final.pdf`。

适用：

- 上市保险公司财务结果分析。
- 新准则/IFRS 17/IFRS 9 影响分析。
- 金融机构经营结果、费用分摊、投资收益、承保贡献拆解。

视觉目标：

- 专业财务结果盘点。
- 白底、亮蓝、浅灰、深蓝。
- 图表卡片清晰，正文页高度一致。
- 章节和正文页明显属于同一套模板。

### Mode B: Practice Sharing Deck

对标 `final-20251031 商业银行及保险公司不良处置行业实践分享-1103.pdf`。

适用：

- 不良处置行业经验分享。
- 监管培训、案例复盘、流程合规、资产处置观察。
- 银行/保险风险案例讲解。

视觉目标：

- 培训分享感更强。
- 白底、书法感亮蓝标题、建筑照片拼贴。
- 文字和案例密度高，但通过蓝线、卡片、时间线组织阅读路径。

### Mode C: Global Policy Comparison

对标 `global_loan_writeoff_policy_comparison_20260121015942.pdf`。

适用：

- 全球/区域监管政策、司法辖区、会计准则、税务协同和制度比较。
- 贷款核销、不良处置、拨备、资本约束、跨境监管协同等制度研究。
- 管理层需要快速理解不同制度模式、监管哲学、行为影响和中国/客户可借鉴点。

视觉目标：

- 白底亮蓝大标题 + 分段横线，形成正式研究报告感。
- 轻量建筑背景图只用于封面、章节或少量政策页底纹，不替代事实证据。
- 大矩阵、2x2 坐标、政策卡片和深色总结页构成主要表达系统。
- 颜色可以比保险财务报告更丰富，但必须低饱和、语义稳定，不做彩虹表。

### Mode D: Sunong Value Creation Deck

对标 `苏农汇报材料.pptx`。

适用：

- 董事会、管理层或投委会价值创造、经营对标、资本市场视角分析。
- 银行经营质量、ROA/RAROC/EVA/PB、净息差、资产质量、中收转化和资本效率诊断。
- 需要同时承载“深蓝封面仪式感 + 白底高密度分析页 + 模块小结”的正式汇报。

视觉目标：

- 封面使用深蓝渐变、左侧大标题、右侧元数据栏和底部阅读路径。
- 正文页白底、深蓝结论标题、短亮蓝标题线，强调高密度但可读。
- 每个模块以 KPI、截面排名、趋势、桥接图、雷达/散点/回归、模块小结构成节奏。
- 颜色语义服务经营诊断：深蓝为基准/主判断，亮蓝为焦点，红/金只用于风险、压力或约束。

## Insurance Results Visual DNA

### Cover

- 背景：极浅蓝灰渐变。
- 主标题：左上大号深蓝，约 50-60pt，粗黑体。
- 副标题：主标题下方深灰，约 34-40pt。
- 团队和日期：灰色，约 20-24pt。
- 右上：RSM/容诚 logo。
- 中下部：横向金融/数据图片，占画布宽度约 92%、高度约 48%-52%。
- 右侧：竖向 RSM 三色条，灰、绿、亮蓝，靠右边缘。
- 装饰：白色点阵、白色线图/网格，透明度低。

### Section Divider

- 上半部：蓝色玻璃建筑照片，约占画布高度 55%-62%。
- 中部：横向浅灰白标题带，高度约 20%-24%。
- 左侧深蓝编号块：宽约 10%、高与标题带一致，白色大号 `01/02`。
- 章节标题：深蓝粗黑体，约 44-52pt。
- 右侧：细亮蓝竖条。
- 左下：RSM/容诚 logo。
- 右下：版权和年份，小黑字。

### Body Page Chrome

- 页眉高度：约 118px-145px（1920x1080 画布）。
- 左上：RSM 三色短条，靠画布最左边，灰/绿/亮蓝。
- 主 deck title：亮蓝，约 38-44pt，超粗。
- 页眉底：浅灰横带，高约 12-16px。
- 右上：浅蓝模块胶囊，包含模块名和大号章节编号。
- 模块胶囊：左上大圆角，右侧贴边或接近右边缘；右下有亮蓝短条/台阶感。
- 二级页标题：深灰，约 26-32pt，位于页眉下方。

### Takeaway Bar

- 位置：二级标题下方。
- 背景：浅灰 `#F3F3F3`。
- 左侧：亮蓝竖条，宽约 14-18px。
- 正文：黑/深灰，18-21pt。
- 可用 `✓` 符号，但不滥用图标。
- 最多 2-3 行；超过则拆页。

### Main Chart Card

- 背景：白色。
- 圆角：6-10px。
- 阴影：柔和下投影，类似 `0 14px 28px rgba(6,27,58,.12)`。
- 主图卡占正文宽度约 88%-90%。
- 卡片上方留给图表标题；图表标题深蓝/黑色，16-20pt。
- 图表内部背景纯白，不加大面积底色。
- 图表线条：浅灰虚线分隔、低饱和网格。

### Insurance Chart Palette

固定颜色顺序：

1. 亮蓝 `#0099D8`
2. 深蓝 `#061B3A`
3. 中灰 `#666A70`
4. 浅蓝 `#7ED0F0`
5. 负项/辅助灰 `#8A8F95`

不要使用红绿大面积表达业绩好坏。风险或负值可用灰色/深灰，除非用户明确要求风险预警。

### Footer

- 左下：版权或数据来源，灰色，10-12pt。
- 右下：页码，灰色，10-12pt。
- 脚注/口径：主图卡下方左对齐，灰色，10-12pt。

## Practice Sharing Visual DNA

### Cover

- 左侧白色留白，右侧建筑照片。
- 标题：亮蓝书法/楷体风，约 40-48pt。
- 副标题：亮蓝，约 30-36pt。
- 机构和日期：深灰，约 22-26pt。
- 左侧或边缘：亮蓝竖条，旁边可有灰色块。
- 装饰：半透明白色方块拼贴，压在照片和白底交界处。
- 左下：RSM/容诚 logo。

### Agenda

- 左侧约 45%：建筑照片 + 半透明白色几何方块。
- 右侧：目录列表。
- “目录”二字：亮蓝书法体，放在白底标签内，旁边有亮蓝竖条。
- 目录行：编号、标题、页码右对齐。
- 行间使用浅灰分隔线。
- 每行右侧有短浅蓝横线。

### Section Divider

- 左侧或半幅：建筑照片。
- 右侧白底：大号灰色章节编号 + 黑/深灰书法章节名。
- 页面边缘：亮蓝竖条。
- 底部/交界处：白色几何方块拼贴。
- 可加入一句很短的章节问题或章节答案，但不得加入正文 bullet。
- 若无可用图片，使用 `gpt-image-generate` 生成蓝色调金融建筑/办公室/数据屏照片感背景；图片只作为章节视觉，不作为事实证据。

### Body Page Title

- 标题：亮蓝书法/楷体风，约 36-42pt。
- 标题位置：左上，宽度接近全页。
- 标题下方：三段标题线。
  - 左灰线：约 12%-13% 宽。
  - 中深蓝线：约 28%-30% 宽。
  - 右亮蓝长线：剩余宽度。
- 正文 lead：黑色楷体/宋体感，约 17-20pt，行距大。

### Case / Evidence Page

- 主体常用浅灰大证据框。
- 右侧可放淡化金融照片，透明度高，边缘有亮蓝竖条。
- 底部用蓝色图标 + 两列建议文字。
- 案例文字：黑色，楷体/宋体感，18-21pt。
- 重点词：加粗，不额外加红色。

### Two-Case Mechanism Page

- 中部横向亮蓝机制带，高约 18%-22%。
- 左右案例卡：白底、浅蓝描边、蓝色阴影。
- 案例卡顶部可放窄幅照片。
- 案例标签：蓝色圆角胶囊。
- 中心机制：白色胶囊节点，放在蓝色机制带上。
- 中心结论：黑色加粗，约 22-26pt。

### Recommendation Cards

- 三张白色竖卡，圆角较大。
- 卡片顶部：蓝色圆形编号或小进度环。
- 卡片阴影柔和。
- 卡片内文字：楷体/宋体风，约 19-22pt。

## Exactness Rules

为了尽量接近参考 PDF：

- 16:9 画布按 1920x1080 设计。
- 禁止套用深色页眉、紫色渐变、营销风 hero、圆角过大的 SaaS 卡片。
- 正文页背景保持纯白。
- 保险财务正文页不使用书法标题。
- 实践分享正文页不使用保险财务右上模块胶囊。
- 两套风格可以在同一 deck 中共存，但每个章节内必须保持一种视觉模式。

## Asset Rules

- 参考 PDF 的照片效果来自蓝色玻璃建筑、金融数据屏、会议/报表场景。
- 如果没有用户提供可用图片，使用同类金融/建筑图片或生成/检索的替代图片。
- 不要用抽象渐变或 SVG 插画替代照片。
- 照片必须被蓝色调处理，不能是暖色、棕色或强饱和照片。
- 章节页生成图片应记录 prompt、生成工具和使用边界；避免出现可识别公司 logo、真实可读文字和人物特写。

## Global Policy Comparison Visual DNA

### Cover

- 背景：右侧或全幅极浅蓝玻璃建筑照片，整体透明度高，文字区保持白底可读。
- 左侧：一条竖向品牌 rail，亮蓝为主，底部可接深蓝块。
- 标题：亮蓝，约 56-68px，左对齐，允许 2 行。
- 副标题：黑/深灰，约 32-38px；关键范围或“八大司法辖区”等词可亮蓝加粗。
- 说明段：灰色，约 24-28px，控制在 2-3 行。
- 右下：RSM 蓝色小 logo 块或白底 logo；左下放机构和日期。

### Agenda / Research Framework

- 白底，无照片。
- 顶部为亮蓝大标题，下面一条三段横线：灰线、深蓝线、亮蓝长线。
- 主体左右两栏：左侧研究框架/目录，右侧核心维度/方法框架。
- 目录编号使用亮蓝 `01/02/03`，行距大，避免密集目录。
- 底部可放浅灰说明框，说明研究范围、样本和边界。

### Dark Synthesis Page

- 深蓝到亮蓝渐变满版，适用于总结与展望、核心启示、董事会记忆点。
- 顶部居中大标题，白色 46-56px。
- 中部两张半透明深色卡：左为核心启示，右为未来展望或行动含义。
- 底部用 3-4 个大数字形成记忆点；数字必须有解释标签。
- 文字必须保持高对比，正文不低于 24px。

### Policy Philosophy Map

- 主体为 2x2 坐标图，四象限使用低饱和浅色块。
- 坐标轴必须有清晰语义，如监管刚性/税务协同。
- 每个司法辖区用圆点 + 白色标签卡表示；标签不得互相遮挡。
- 底部用 6-8 张类型卡解释每类监管哲学，不要只放点位。

### Policy Framework Cards

- 2x2 卡片，浅灰底，左侧亮蓝竖线。
- 每卡包含图标、卡片标题、机制解释和浅蓝规则框。
- 适用于单一司法辖区的“政策内容与监管框架”页。
- 每卡正文控制 70-110 中文字；超过则拆页。

### Jurisdiction Matrix

- 左侧维度列深蓝/亮蓝，顶部司法辖区列亮蓝。
- 每个司法辖区列使用低饱和浅色底，跨页颜色保持一致。
- 每格只放 1 个主判断 + 1 个短补充；超出则拆成国家章节页。
- 必须有矩阵读法或底部 legend，避免成为事实堆放。

## Sunong Value Creation Visual DNA

### Dark Cover

- 背景：深蓝到中蓝渐变，不放照片；整体安静、董事会汇报感。
- 左上：短亮蓝线 + 小字场景标签，如“董事会汇报”。
- 右上：白底 RSM logo 方块。
- 左中：客户名/股票代码，小号浅蓝灰。
- 主标题：白色、超大、轻字重，允许 2-3 行换行。
- 副标题：浅蓝灰、加粗。
- 正文说明：浅色，保留若干黄色关键字，说明框架和结论。
- 右侧：元数据竖栏，包含报告期、口径、对象、样本、数据来源。
- 底部：阅读路径 breadcrumb；左下页码。

### White Diagnostic Body

- 白底，无厚重页眉。
- 左上短亮蓝线，下面深蓝或黑色结论标题。
- 副标题说明样本、口径、方法和本页证明对象。
- 主体优先采用左 65%-70% 主图 + 右 25%-30% 诊断读法；或上主图、下 3 个解读卡。
- 每页必须有明确 benchmark 或排名方向，服务经营诊断。
- 来源条左下，页码弱化；不得遮挡图表。

### KPI Dashboard

- 用于执行摘要、指标全景和模块开篇。
- 4 张 KPI 卡为典型结构；每卡包含指标编号、指标名、大数字、排名/中位数、解释句。
- 指标卡顶部用亮蓝/红/金表达状态，不做大面积红绿。
- 中下部必须有经营视角和资本市场视角两块综合结论，避免 KPI 列表化。

### Module Summary

- 用于每个经营模块结尾。
- 标题采用 `Pxx · 模块小结 · 模块名` 或等价结构。
- 主体通常为 3 个结论卡 + 证据数字/回链页 + 下一模块提示。
- 不得只有空泛三卡；每张卡必须回链前文证据页或关键数字。

### Dense Appendix / Backup Pages

- 苏农材料中存在高密度备份页，允许更密，但必须标为附录或补充页。
- 即使是备份页，也必须保留结论标题、来源、页码和清晰分组。
- 不得把备份页字号和密度带入正文核心页。
