# Stress Scenario Content Standard

用于生成压力情景、敏感性分析、财务推演和“两种图景”页面。适用于银行、保险、资产估值、交易结构、投委会、董事会和风险审查材料。

## Core Principle

压力情景不是装饰性表格。每个情景都必须说明：假设如何变化、影响哪些核心指标、客户应如何解读，以及触发什么行动。

## Scenario Naming

优先使用中性名称：

- `基线情景`
- `温和压力`
- `中度压力`
- `极端压力`
- `行动后情景`

避免：

- `乐观 / 中性 / 悲观`，除非用户或模型已有明确命名。
- `最好 / 最坏`，除非是严格上下界测算。

## Parameter Axes

标准压力情景应包含 2-3 个参数轴，每轴 3 档：

| 场景 | 常用参数轴 |
|---|---|
| 银行经营 | 净息差下降、风险成本上升、贷款增速放缓 |
| 保险经营 | 投资收益率下行、赔付率上行、费用率上行 |
| 资产估值 | 出租率下降、租金下调、退出 cap rate 上升 |
| 交易结构 | 回款延迟、违约率上升、处置折扣扩大 |
| 政策/合规 | 执行时间延后、适用范围收窄、资本/税务成本上升 |

## Output Indicators

输出指标必须与执行摘要或核心 KPI 对齐。不要在压力页突然引入新的核心指标。

示例：

- 银行：ROA、核心一级资本充足率、不良率、PB 修复条件。
- 保险：净利润、综合收益、偿付能力充足率、承保贡献。
- 资产估值：IRR、DSCR、净现值、可接受收购价。

## Page Types

### Stress Scenario Table

适用于 3-4 个离散情景比较：

- 左侧：情景名称。
- 中部：关键假设。
- 右侧：输出指标。
- 最右：管理含义。

### Sensitivity Matrix

适用于两个变量连续变化：

- 行变量和列变量必须有单位。
- 基准格必须高亮。
- 安全区、观察区、风险区使用低饱和色，不用刺眼红绿。

### Two-Scenario Future Page

适用于“若行动 / 若不行动”或“基线 / 行动后”：

- 不只展示两个独立数字，必须展示 `delta`。
- 必须说明 delta 来自哪个行动或假设。
- 每个图景最多 3 个指标和 2 条解释。

## Required Schema

```json
{
  "scenario_analysis": {
    "scenario_names": ["基线情景", "温和压力", "极端压力"],
    "parameter_axes": [
      {"name": "净息差下降", "unit": "bp", "levels": [0, -10, -25]},
      {"name": "不良率上升", "unit": "bp", "levels": [0, 20, 50]}
    ],
    "output_indicators": ["ROA", "CET1", "NPL"],
    "baseline_case": "基线情景",
    "management_implication": "若进入极端压力，需优先压降高资本消耗资产并提高风险定价"
  }
}
```

## Quality Rules

- 假设、公式、输出指标和来源必须可追溯。
- 情景参数不得凭空设定；如为项目组假设，明确写“RSM 情景假设”。
- 压力情景不能替代预测；标题应写成“在 X 假设下，Y 指标承压”，不要写成“未来必然下降”。
- 极端压力不等于危言耸听，必须可解释、可复算、可落行动。
