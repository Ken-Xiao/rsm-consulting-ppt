# Professional Chart Rulebook

用于明确金融咨询 PPT 中图表的专业标准。任何核心图表、客户交付图表、合伙人审阅图表都必须满足本规则。先用 `chart-decision-tree.md` 选型，再用 `financial-chart-grammar.md` 和本文件检查专业度。

## Chart Contract

每张核心图表必须在 `chart_data/Pxx.json` 中记录：

```json
{
  "chart_id": "chart_P06_01",
  "chart_type": "stacked_contribution_split",
  "chart_point_of_view": "利润改善主要来自投资端，承保贡献未同步改善",
  "claim_title_link": "P06",
  "conclusion_id": "c_001",
  "benchmark_type": "2025 vs 2024",
  "comparison_basis": "同一公司、同一披露口径、同一期间",
  "focus_series": ["投资收益贡献"],
  "annotation_priority": ["投资收益提升", "承保贡献下降"],
  "unit": "人民币亿元",
  "period": "2024-2025",
  "sample": "7 家上市险企",
  "source": "公司年报、项目组整理",
  "lineage_ids": ["fact_014", "fact_015"],
  "chart_readout": "净利润增长不能直接等同于承保质量改善"
}
```

缺少 `chart_point_of_view`、`benchmark_type`、`unit`、`period`、`sample`、`source`、`lineage_ids` 的图表，不得作为核心页主证据。

## Professional Annotation Rules

- 每张图最多 2 个 callout，优先标注结论相关的拐点、差异、异常值或阈值。
- 标注必须靠近数据点或分项，不放在远离图表的解释段落里。
- 标注文字用“事实 + 含义”，例如：`投资收益贡献提升 12pct，利润改善更依赖资本市场表现`。
- 不标注所有数据点；只标注帮助读者读懂页面标题的点。
- 图表中若有焦点主体，必须用深蓝或亮蓝；其他主体用浅灰/浅蓝。

## Benchmark Rules

| Chart task | Required benchmark |
|---|---|
| 同业对比 | 同业中位、均值、头部样本或监管阈值 |
| 时间趋势 | 起始年份、拐点年份、政策/准则切换点 |
| 双年度对比 | 同一样本、同一口径或明确口径差异 |
| 估值/价格判断 | 市场可比、交易可比、敏感性区间或底线阈值 |
| 风险评分 | 评分口径、阈值、风险等级解释 |
| 行动路径 | 基线、不行动、推荐路径三种情景 |

没有基准的图表通常只能展示事实，不能支撑强咨询判断。

## Chart-Type Professional Rules

### Ranking Bar

- 必须注明排名方向：高者优 / 低者优。
- 焦点主体最多 2 个。
- 必须显示中位线、监管线、目标线或头部样本之一。
- 样本少于 4 个时不要写“排名”，改写为“对比”。

### Trend Line

- 历史与预测必须区分线型或底色。
- 政策、准则、会计口径变化必须加垂直标记。
- 单页超过 4 条线必须拆页或改为分面图。
- 不同口径指标不能强行放在一张趋势图里。

### Stacked Contribution

- 用于构成贡献，不用于强因果证明。
- 颜色顺序跨页固定；同一分项不能在不同页变色。
- 小于 5% 的分项优先合并或移到脚注。
- 正负项混合时优先改为 waterfall。

### Waterfall / Bridge

- 起点和终点必须清楚，且单位一致。
- 增减项按业务传导顺序排列，不按数值大小机械排序。
- “其他”不得超过总变化的 25%；否则拆分。
- 每个增减项必须能回链来源或计算公式。

### Scatter / Regression

- 样本少于 5 个不得画回归线。
- 回归关系不得写成因果关系。
- R²、显著性或拟合解释不足时，只能说“相关性不明显/需要进一步验证”。
- 焦点样本必须直接标注名称。

### Sensitivity / Heatmap

- 必须高亮基准情景。
- 色阶最多 5 档；不要用红绿作为唯一编码。
- 行列变量必须有单位和取值范围。
- 热力图旁必须写“安全区间/风险区间/决策含义”。

### Financial Table

- 表格不是数据堆放；每张表必须有排序、分组、重点高亮或小结行。
- 表格首列和表头必须冻结式视觉强化。
- 金额、百分比、bp、倍数不能混在同一列。
- 负数、缺失值、不可比项必须有统一符号说明。

## Color And Label Rules

- 默认色板：亮蓝 `#0099D8`、深蓝 `#061B3A`、中灰 `#6A6F76`、浅蓝 `#7ED0F0`。
- 风险红只用于重大风险、负值或不可行，不用于普通差异。
- 同一指标/分项跨页颜色必须一致。
- 直接标签优先于图例；图例超过 5 项说明图表过复杂。
- 数据标签只保留关键标签；全部标签导致拥挤时改为表格或分面图。

## QA Before Delivery

- 图表能否单独证明页面标题？
- 图表观点是否与 `conclusion_evidence_matrix` 中的结论一致？
- 基准是否清楚？
- 口径变化是否说明？
- 标注是否服务结论而非装饰？
- 图表在 contact sheet 下是否仍能识别结构？
- 来源和数据血缘是否完整？
