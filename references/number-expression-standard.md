# Number Expression Standard

用于统一中文金融咨询 PPT 中的数字、金额、百分比、同比/环比、基点和估值表达。写标题、副标题、图表标签、结论条、注释和来源时必须遵守。

## Core Rules

1. **金额单位优先用“亿元 / 万元”**  
   客户汇报中金额默认保留 1 位小数；若原始披露为整数且无需精度，保留整数。  
   - 推荐：`保费收入 1,236.4 亿元`、`处置成本 8,450 万元`
   - 避免：`123640000000 元`

2. **百分比用 `%`，变化用 `pct` 或 `bp`**  
   - 比率本身：`ROE 为 9.8%`
   - 百分点变化：`同比提升 1.2 pct`
   - 利率/息差/收益率变化：`下降 18 bp`
   - 避免：`同比提升 1.2%`（若实际是百分点）

3. **同比/环比/较期初统一写法**  
   - `同比+8.4%`
   - `环比-2.1%`
   - `较期初+3.5 pct`
   - `较同业中位低 42 bp`

4. **排名必须写清样本口径**  
   - 推荐：`样本 7 家中排名第 2`
   - 推荐：`9 家上市农商行中位数为 0.76%`
   - 避免：`排名靠前`、`表现领先`

5. **估值和倍数保留 1-2 位小数**  
   - `PB 0.62x`
   - `PE 8.4x`
   - `EV/EBITDA 6.8x`
   - 若是区间：`0.55x-0.70x`

6. **增长率、利润率、费率、成本率默认保留 1 位小数**  
   只有当变化很小或金融监管口径要求时保留 2 位小数。

7. **基点优先用于利率、息差、收益率、信用成本**  
   - `净息差收窄 16 bp`
   - `信用成本上行 22 bp`
   - 不要写成 `0.16个百分点`，除非用户材料原文要求。

8. **负数表达避免歧义**  
   - 图表标签：`-12.4%`
   - 文字：`同比下降 12.4%`
   - 不写：`同比-下降 12.4%`

9. **区间和情景必须写明前提**  
   - `在基准情景下 IRR 为 7.8%`
   - `若空置率上行至 18%，项目 IRR 降至 5.2%`

10. **标题中的数字不宜超过 2 个**  
    标题可以放关键数字，但不要把标题写成数据表。复杂数字放入副标题、结论条或图表标签。

## Chinese Before / After

| Weak | Client-ready |
|---|---|
| `公司利润增长较好` | `归母净利润同比+8.4%，但承保利润贡献仍低于样本中位` |
| `费用率有所改善` | `费用率同比下降 1.6 pct，主要来自销售费用压降` |
| `净息差压力比较大` | `净息差较上年收窄 18 bp，资产端收益下行未被负债成本下降完全对冲` |
| `估值仍然偏低` | `PB 仅 0.62x，低于样本中位 0.18x，市场仍未充分定价盈利修复` |
| `建议关注资产质量` | `建议以关注类迁徙率和逾期 90+ 占比作为资产质量月度预警指标` |

## Label Rules

- 图表坐标轴必须有单位：`亿元`、`%`、`bp`、`x`。
- 图例尽量用主体名称或指标名称，不使用 `Series 1 / Series 2`。
- 表格数值列标题写清单位，避免每个单元格重复单位。
- 同一页同一指标的小数位必须一致。

## QA Checklist

- 是否把百分比变化和百分点变化混用。
- 是否说明排名样本、期间和口径。
- 是否存在“明显、大幅、较好”等没有数字支撑的形容词。
- 是否同一页同一指标小数位不一致。
- 是否将情景测算输出写成事实判断。
- 是否每个核心数字能回链到 `lineage_map.json`、`data_pool.json`、`chart_data/` 或原始材料。
